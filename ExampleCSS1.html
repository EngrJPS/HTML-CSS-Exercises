Formatting to the HTML elements

In this chapter, we start with an overview of CSS concepts and CSS basic syntax. We put those things into practice by applying CSS rules to various elements, including span and div elements. We show you how to position those rules (1) the top of the web page's main file or (2) in an external file.

CSS Rules

The way CSS works is that CSS rules are applied to elements within a web page. Browsers determine which elements to apply the CSS rules to with the help of Selectors. There are quite a few different types of selectors. For now, we'll introduce type selectors and universal selectors.

Type Selectors

With type selector, you use an element type (e.g, hr, h1, p) to match all instances of that element type and then apply specified formatting features to those instance.

For Example:

        
             
    <!-- The following CSS rule uses a type selector 
    with the hr element type and applies a width of 50% 
    to all the hr elements in the current web page -->
            
            hr {width: 50%}

Universal selectors

Now for another type of selector ‐ the universal selector. The universal selector uses the same syntax as the type selector, except that instead of specifying an element you specify *. The asterisk is a wildcard. In general, a wildcard is something that matches every item in a collection of things. 

        
            
    <!-- The * matches every element in a web
        pages collection of elements.-->
            
         * {font-weight: bold}

CSS Syntax and Style

CSS Syntax

In this section, we address CSS syntax details. First ‐ the syntax style container. Refer from the previous lessons and note how the three CSS rules are enclosed in a style container.


Here's the relevant code:

        
            <style>
                * {text-align: center;}
                hr {width: 50%;}
                h2, p {font-style: italic; color: blue;}
            </style>

In the style start tag, it's legal to include a type attribute with a value of "text/css", like this: 

        
            <style type="text/css">

CSS Style

Now we'll look at some CSS guidelines that are not enforced by browsers or HTML5 standard. They are style guidelines, and you should follow them so your code is easy to understand and maintain.

For short CSS rules, use this format:
